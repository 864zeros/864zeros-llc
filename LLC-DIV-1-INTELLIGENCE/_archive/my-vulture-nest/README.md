# 🦅 The 864z Vulture Nest

This repository houses the core protocol and output for the **864z Scout Agent**, an autonomous system designed to identify "Successfully Neglected" digital gaps in the market.

## 🎯 Project Goal
The Vulture Nest's mission is to systematically discover high-potential software opportunities (Mobile Apps, Micro-SaaS Web Apps, and Chrome Extensions) by finding critical "wounds" (user frustrations) within existing, popular applications (the "Carcasses").

## 📝 Core Protocol: The "Carcass-First" & "Sweet Spot" Strategy
The agent operates under a strict protocol defined in `gemini.md`. This protocol ensures:

1.  **Carcass-First Scouting:** Opportunities are identified by first finding an *existing, successful application* (the "carcass") within our target categories.
2.  **"864z Sweet Spot" Filtering:** We focus exclusively on projects of small to medium size (XS, S, M), enabling high-velocity builds and avoiding large, enterprise-scale solutions (Size L).
3.  **"Blood in the Water" Validation:** Every opportunity is driven by verifiable user frustration and unmet needs, ensuring a validated market demand.

## 🚀 How to Run the Autonomous Vulture Nest Cycle
To launch the 864z Scout Agent and begin identifying new opportunities, follow these two simple steps in your Gemini CLI session:

1.  **Assimilate Protocol:** Instruct the agent to load its core directives:
    ```
    Read, understand, and fully assimilate the protocol defined in `gemini.md`. This document is my core directive and contains all operational parameters, including the 'Carcass-First' strategy, the '864z Sweet Spot' filter that avoids large enterprise platforms, and the XS/S/M/L project sizing chart. All your actions must strictly adhere to this protocol.
    ```

2.  **Initiate Autonomous Cycle:** Once the agent confirms assimilation, give the command to start scouting:
    ```
    VULTURE_NEST: RUN AUTONOMOUS CYCLE
    ```

## 📁 Key Files

*   **`gemini.md`**: The brain of the operation. Contains the complete, up-to-date protocol for the 864z Scout Agent.
*   **`Vulture_Nest.md`**: The output catalog. All identified and validated opportunities will be appended to this file.
*   `.env`: (Optional) Contains your `APIFY_TOKEN` for web scraping, if required by the agent.
*   `requirements.txt`: (Optional) Lists Python dependencies, should the agent decide to generate and execute local Python scripts.
*   \documentation\instructions.txt
